# ABAC (Attribute-Based Access Control)
## I. 속성(태그) 기반의 유연한 접근 제어, ABAC 개념
:::tip [개념]
사용자, 리소스, 권한, 환경등 다양한 속성(또는 태그)을 판단하여 접근을 제어를 하는 기법
:::
ㅡ RBAC에서 발전해 <u>유연하게 확장 가능한 접근 제어</u>가 가능한 모델
ㅡ 제로트러스트 구현


## II. 클라우드 환경의 AC에 적합한 ABAC 매커니즘
### 가. ABAC 도식화
![](./ABAC-00.png)
ㅡ 사용자에게 할당된 정책 변경을 통해서 유연하게 접근 제어 가능

### 나. ABAC 상세 기능 설명
| 구분 | 설명 | 주체 |
|---|---|---|
| (1) 속성(태그) | - 접근을 결정하는 판단 요소 | - 리소스<br/> - 사용자<br/> - 환경 |
| (2) 사용자<br/>(주체) | - 객체에 접근을 시도하는 사용자 또는 시스템 | - Staff <br/>- Administrator <br/>- Sysop |
| (3) 행동 | - 접근 요청이 이루어지는 행위 | - CRUD<br/> - 인증<br/>   |
| (4) 정책 | - 접근 허용 또는 거부되는 조건 정의 및 관리 | - XACML<br/> - PBAC |
| (5) 객체 | - 접근 대상이 되는 엔티티 | - DB<br/> - 파일<br/>  |

ㅡ 세밀한 접근 제어를 통해 제로트러스트 아키텍처 구현 가능
